フロントエンドエンジニアになってから7年経ちました！
この7年間を振り返って、今からフロントエンドエンジニアやりたいと思っている人向けに、
最初からやるべきだった（自分の反省点）とやって良かったことをシェアしたいと思います。

まず、やって良かったことから行きます。

### なんでエンジニアになりたいか？きちんと答える
なんでエンジニアになりたいか、ここを曖昧にしたまま走ると **8割くらいの確率で挫折** します。

自分が通っていたプログラミングスクールでもたくさんの人と知り合いましたが、  
今でもエンジニアを続けている人は数名です（笑）

自分の答えは当時から変わっていません。

> 「物作りが大好き！」

これがあったから、

- エラーで詰んでも
- 意味不明なバグで3時間溶けても
- 新しい技術が出てきても

続けられました。

結局、エンジニアって **勉強し続ける仕事** です。

「稼げそう」「リモートできそう」みたいな理由だけだと、長期戦でかなりキツい。  
逆に「作るの好き」がある人は強いです。

### 知らない前提で行く
初心者の時ほど、変に知ったかぶりしない方が成長早いです。

自分は最初、

「なんとなく理解した気になる」  
↓  
「実際に作ると何も分からない」

を何回も繰り返しました。

特にJavaScript。

チュートリアル見た瞬間は理解した気になるのに、  
自分で作ろうとすると手が止まる（笑）

でもそれ普通です。

むしろ、

- 「分からない」
- 「知らない」

を前提に質問できる人の方が伸びます。

あと、**公式ドキュメント読む癖** は本当に大事。

最初は意味不明でも、実務経験が増えると急に読めるようになります。  
MDN Web Docs は特に神でした。

### 手を動かして作る
動画だけ見ても、ほぼ身につきません。

実際にコードを書くと、「理解したつもり」が一瞬で壊れます（笑）

自分はポートフォリオ作った時に、初めて本当に勉強になった感覚がありました。

小さいものでもいいので、たとえば

- Todoアプリ
- 天気アプリ
- 模写コーディング
- 自分用の管理ツール

とかを作った方が圧倒的に成長します。

フロントエンドは特に「見える成果物」があるので、作れば作るほど楽しくなる。

## 反省点: やるべきだったこと

### フロントとは言えバックエンドの基礎を勉強すべき
これ、本当に痛感しています。

最初の頃は、

- React
- CSS
- UI
- アニメーション

ばかりやってました。

でも実務入ると結局、

- API
- DB
- 認証
- セキュリティ
- HTTP
- キャッシュ

この辺の理解が必要になります。

フロントだけ分かってても、「なんでこのエラー起きてるの？」が理解できない。  
逆にバックエンド知識が少しあるだけで、実務の理解度が一気に変わります。

別にバックエンドエンジニアレベルを目指さなくてOKです。

でも、

- HTTPの仕組み
- REST API
- JSON
- Cookie / Session
- JWT
- SQLの基礎

この辺は早めに触った方が絶対いいです。

フロントエンドは「画面作るだけの仕事」ではないので。

### SIMPLE IS BEST
自分は元々、絵を描くのが大好きで、少女漫画みたいな凝ったデザインが大好物です。  
だから最初は、

- 超複雑なUI
- 無駄に動くアニメーション
- 謎に凝った設計

をやりがちでした（笑）

でも実務やると分かる。  
**シンプルなUIの方が強い。**

理由は単純で、

- ユーザーが迷わない
- 保守しやすい
- バグ減る
- 実装速い
- チーム開発しやすい

からです。

コードも同じ。

初心者ほど難しい設計したくなるけど、実際に評価されるのは **読みやすいコード** です。  
未来の自分や他人が読めるコードを書く。これ超大事。

### フレームワークより基礎
これも遠回りしたポイント。

最初、Reactだけ覚えればいいと思ってました。

でも結局、

- HTML
- CSS
- JavaScript

の基礎が弱いと詰みます。

基礎理解してない状態でフレームワーク触ると、ただの「コピペ職人」になります。

逆に基礎が強い人は、ReactでもVueでもNext.jsでもキャッチアップ速いです。  
流行りは変わるけど、基礎はほぼ消えません。

## 最後に
エンジニアって「才能ある人だけの仕事」みたいに見えることあるんですが、実際は、

- コツコツ続けた人
- 分からないを放置しなかった人
- 手を動かし続けた人

が残ります。

自分も最初はかなり苦戦しました。  
でも7年続けて思うのは、完璧じゃなくても、ちゃんと積み上がるということです。

これから始める人の参考になれば嬉しいです 🙌
